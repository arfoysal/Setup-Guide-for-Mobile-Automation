Java JDK Download, install, set JAVA_HOME and path in environment‚Äôs system variable.

  • Download and install Java JDK any version you want from java 8 or above https://www.oracle.com/java/technologies/downloads/
  • Open Environment Variables and click on the new System variable ‚Üí set Variable name: JAVA_HOME Variable value: Your Java SDK Location
  • Select Path system variables then click on Edit ‚Üí click on new and write %JAVA_HOME%\bin
  • Test successful Java setup ‚Üí Open CMD and write java -version

Maven zip file Download, extract set MAVEN_HOME and path in environment‚Äôs system variable.

  • Download and extract the Maven Binary zip file inside your Java folder(optional) ‚Üí to keep everything organized https://maven.apache.org/download.cgi
  • Open Environment Variables and click new System variable ‚Üí set Variable name: MAVEN_HOME Variable value: Your Maven directory
  • Select Path system variables then click on Edit ‚Üí click new and write %MAVEN_HOME%\bin
  • Test successful Maven setup ‚Üí Open CMD and write mvn -version

Node.js Download, install, & set path in environment‚Äôs system variable.

  • Download and install Node JS ‚Üí it will automatically set system environment required path https://nodejs.org/en/
  • (Optional) Select Path system variables then click on Edit ‚Üí click on new and write C:\Program Files\nodejs\node_modules\npm\bin
  • Test successful Node JS /npm setup ‚Üí Open CMD and write node -v npm -v

Android Studio Download, install, set ANDROID_HOME and path in environment‚Äôs system variable.

  • Download and install Android Studio https://developer.android.com/studio
  • Open Environment Variables and click on the new System variable ‚Üí set Variable name: ANDROID_HOME Variable value: Your Android SDK Folder Location
  • Select Path system variables then click on Edit ‚Üí click on the new and add this four path variables and @important make sure they remain in this order %ANDROID_HOME%\emulator %ANDROID_HOME%\tools %ANDROID_HOME%\platform-tools %ANDROID_HOME%\tools\bin

Android Studio SDK Manager Setup

  • Open Android SDK manager from the Tool tab
  • Select SDK Platforms ‚Üí Checked & installed ‚ÄúAndroid API 32‚Äù and at least one Android version
  • Select SDK Tools ‚Üí Uncheck Hide Obsolete Packages ‚áí Checked & installed 1. Android SDK Build-tools 32 2. Android Emulator 3. Android SDK Platform-Tools 4. Android SDK Platform-Tools(Obsolete) 5. Intel x86 Emulator Accelerator (HAXM installer)
  • Select SDK Update site ‚Üí make sure all listed items are Enabled

Android Studio AVD(Android Virtual Devices) Manager Setup / Emulator

  • Open Android AVD Manager from the Tool tab
  • Click on click Create Virtual Device ‚áí <Choose a device definition / a model > ‚áí Next ‚áí <Select system Image/ android version> ‚áí Next ‚Üí provide ADV/emulator name > Finish
  • Done! you just create Android Virtual Device
  • From the Actions column, you can perform Launch, Edit, Duplicate, Wipe data, Cold boot now, Delete .etc on your selected AVD device.

Android Debug Bridge(adb) ‚Üí udid | install |PackageName |appActivity

To show connected/active adb devices with their Unique Device Identifier(udid)

adb devices

Install APK ‚áí Many ways! you can do that

‚Üí Drag and Drop the apk file inside the emulator ‚Üí Install using play store ‚Üí Download apk file using emulator browser than install ‚Üí transfer apk file to emulator storage location than install ‚Üí Install using cmd by the helping of adb(Android Debug Bridge)

# if only one adb device connected to your computer 
adb install "path to .apk"
# if multiple device are connected 
adb -s "device-udid" install "path to .apk"  #-s source/serialNumber/Identifier
# optional options are 
# adb install [options] path -!
#-r: Reinstall an existing app, keeping its data.
#-f: Install package on the internal system memory.
#-d: Allow version code downgrade.
#-g: Grant all permissions listed in the app manifest."
# https://developer.android.com/studio/command-line/adb

Emulator ‚Üí List | Launch | Shutdown | Cold boot | Wipe data

  • To show List of available emulator names
    emulator -list-avds
  • Launch/start/Open an Emulator ‚Üí after start emulator it will keep alive until close this current CMD window.
    emulator -avd emulator_Name
# OR
emulator @emulator_Name
  • Close/Shutdown an Emulator ‚Üí Many ways are available

    ‚Üí In CMD window simply click Ctrl + C and wait for few seconds ‚Üí it will close the emulator.

    ‚Üí By clicking close button in emulator UI ‚Üí Closing the CMD window

  • Start Emulator in Cold Boot view
    emulator -avd emulator_Name -no-snapshot-load
# OR
emulator @emulator_Name -no-snapshot-load
  • Wipe-data/RESET an Emulator 
    emulator -avd emulator_Name -wipe-data
# OR
emulator @emulator_Name -wipe-data

Appium inspector - it's an advanced UI inspector for mobile OS

  • Run Appium Inspector

    Run Appium Inspector by clicking on and set Remote Path ‚Üí /wd/hub

  • Desired Capabilities
    • To start a session ‚Üí with an only connected or running adb device if multiple devices are connected then it will start with priority one.
      {
  "platformName": "android"
}
    • To start a session with a specific physical or emulator device 
      {
  "platformName": "android",
	"udid" : "Your device or emolator udid"
}
    • To start a session with a specific physical or emulator device and a specific app
      {
  "appium:deviceName": "Iblish",
  "platformName": "android",
  "appPackage": "Your app package name",
  "appActivity": "Your app Activity name"
}
    • To start a session with a specific device & app, and specific automation engine
      {
  "appium:deviceName": "Iblish",
  "platformName": "android",
	"automationName": "UiAutomator2",
  "appPackage": "Your app package name",
  "appActivity": "Your app Activity name"
}
    • For more Appium Desired Capabilities
